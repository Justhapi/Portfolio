Overview

A platform that works — but doesn't feel like it

The platform's backend features functioned. The UI didn't — it failed to serve either student or professor mental models, and had no room for the new features the engineering team was already planning.

"How might we design a UI that serves both student and professor workflows while leaving room for new feature growth?"

01 — Research

Evaluating both sides of the platform

I explored the platform from both a student and professor account, then followed up with user interviews to understand what each group actually needed.

Key Findings

  • Students needed clearer project status visibility — current state felt opaque
  • Professors needed faster ways to review and respond to student applications
  • Both groups wanted notifications that differentiated urgent vs. routine updates
  • Role-switching between accounts lacked clear visual distinction

02 — Design

Designing for today and leaving room for tomorrow

Mockups were designed to accommodate both current features and the planned implementations the engineering team would build — so the UI wouldn't need to be rethought when new features landed.

Key Design Decisions

  • Purdue brand gold as the primary action color — reinforces institutional identity
  • Card-based project board for professors — scannable at a glance
  • Persistent role indicator in the nav header — reduces account-switching confusion
  • "Coming soon" treatment for unbuilt features — signals growth without breaking current layouts

03 — Delivery

From design to frontend

High-fidelity mockups were shared with engineering in Figma. Weekly syncs ensured specs translated accurately. I then transitioned into frontend development support to help ship the redesigned interface.

04 — Reflections

Designing within constraints sharpens your instincts

  • Brand color accessibility — working within Purdue's palette required careful contrast management; initial choices failed checks and needed revision
  • Figma at scale — component naming and variant structure dramatically affect handoff quality
  • Design → engineering — transitioning to frontend gave me a clearer mental model of implementation constraints that now shapes how I spec every design
