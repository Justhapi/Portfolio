Overview

Journey maps go stale. Nobody notices.

Journey maps are living documents — but as data grows and time passes, touchpoints become outdated without notifying the manager. The result: decisions made on maps that no longer reflect reality.

"How might we help journey map managers maintain accurate maps without adding to their workload?"

01 — Research

What managers actually need

Research ran in two tracks — competitive and interaction analysis to understand the domain, plus desk research and user interviews to understand the managers themselves.

Key Findings

  • Managers spend significant time manually cross-checking data sources for relevance
  • Outdated content goes undetected until a major review cycle
  • Managers want flagging — not auto-editing — to keep control over map integrity
  • Integration must feel native, not bolted on

02 — Design

The agent surfaces staleness. The manager decides.

"Control stays with the manager. The agent informs — it doesn't act."

Data monitoring

Continuously watches connected data sources for changes relevant to mapped touchpoints.

Staleness flagging

Highlights journey map nodes where underlying data has shifted beyond a threshold.

Update suggestions

Proposes wording or sentiment changes for manager review — never applying them automatically.

Audit trail

Logs every flag and manager decision for accountability and future model training.

Sponsor Feedback

  • Notification placement moved from modal to inline badge — less interruptive
  • Confidence scoring added so managers can prioritize which flags to review first
  • Bulk-review mode added for high-volume map managers

04 — Reflections

Trust is built through transparency, not efficiency

Early concepts had the agent auto-applying low-confidence changes. Every user in testing rejected this. The most important design decision was keeping the manager in control — and making that control feel natural, not like a safety net.

Co-leading across design and engineering also showed me how much clarity of spec matters when the two disciplines need to move fast together.
